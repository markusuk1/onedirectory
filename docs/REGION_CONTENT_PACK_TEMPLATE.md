# Region Content Pack (Template)

Use this as the *source of truth* for generating region-specific Guides + homepage copy without thin/duplicated content.

## 1) Region identity
- **siteId:** (e.g. `northwest`)
- **shortName:** (e.g. `North West`)
- **region label (human):** (e.g. `North West England`)
- **primary city anchor:** (e.g. `Manchester`)
- **secondary city anchors:** (e.g. `Liverpool`, `Preston`, `Chester`)

## 2) Transport anchors (for minibus/coach guides)
### Airports
- **primary airport:** (e.g. Manchester Airport)
- **secondary airports:** (optional)

### Rail hubs (top 3–8)
- Main stations / interchanges, per city.

### Coach/minibus access notes (local constraints)
- City-centre access/parking realities
- Clean Air Zones / LEZ / ULEZ style constraints (if applicable)
- Common congestion/closure patterns (event nights, football matchdays)

## 3) Nightlife anchors (for party bus / nights out)
For each major city area include:
- **area name** (e.g. Northern Quarter)
- **why it matters** (pickup hotspots, late-night crowds)
- **typical drop-off pattern** (1–3 sentence practical note)

## 4) Events & venues (for event transport checklist + evergreen guides)
### Stadiums / arenas (top 5–12)
- Name + city

### Major annual events / festivals (top 5–12)
- Event name + typical season/month (no exact dates needed)

### Wedding / conference venues (top 5–15)
- Name + area

## 5) Geography / coverage (for relatedLocations)
- **Top towns/areas (SEO + related locations):** list 10–30
- **Rural / coastal notes:** where journeys become longer / routes less direct

## 6) Service-specific anchors (for other verticals)
### Van hire
- Typical use-cases unique to region (trade patterns, student moves, etc.)
- Any local restrictions for larger vehicles (city centres)

### Skip hire
- Council permit realities: common constraints, contact expectations
- Typical property types (terraces, driveways, narrow streets)

### Locksmith
- Common property/lock types in region (uPVC prevalence, multi-point locks)
- Local emergency expectations (urban vs rural response considerations)

## 7) Internal linking map (avoid duplication)
- 5–15 **must-link** location slugs per region (from your location dataset)
- 3–8 **must-link** guide slugs per region
- 1–4 **must-link** product landing pages

## 8) Core Guide Pack (per region)

Minibus/coach guide set to generate for each region (minimum viable set):
- Party bus (primary city)
- Self-drive minibus (licence + local pickup patterns)
- 16-seater minibus hire (use-cases + local examples)
- Airport transfers (region airports + hubs)
- Wedding transport (venues + logistics)
- How to choose an operator (region-specific constraints)
- Pricing explained (region-specific constraints + common mismatches)
- Airport transfer planning (region airports + rail hubs)

Plus utility guides (site-aware, but can be expanded):
- Operator checklist
- Pricing explained (utility)
- Event transport checklist
- Travel resources

Optional verticals per region:
- Van hire guides
- Skip hire guides
- Locksmith guides

## 9) “Uniqueness rules” (anti-thin-content)
Every region’s guide should include:
- **At least 2** named local anchors (venues/areas/hubs)
- **At least 1** local constraint (access/permits/congestion)
- **At least 1** region-only checklist item or planning tip
- **FAQs** rewritten with region context (not swapped nouns)
